Callback

Un callback es una función que se pasa como argumento a otra función y se ejecuta en un momento específico dentro de esa función. Este mecanismo es crucial en la programación, especialmente para manejar eventos asíncronos como solicitudes de red o temporizadores, sin bloquear el flujo principal del programa. Por ejemplo, en JavaScript, las funciones de callback se utilizan frecuentemente con métodos como setTimeout o al manejar eventos. En Python, se emplean en bibliotecas de interfaces gráficas y redes. Las callbacks permiten que el código sea más modular y reutilizable, facilitando la personalización del comportamiento de funciones genéricas sin necesidad de duplicar código.

Promiss

Una promesa (promise) es un objeto en programación que representa la eventual finalización o falla de una operación asíncrona y su valor resultante. Permite manejar operaciones asíncronas de manera más ordenada y legible comparado con callbacks. Una promesa puede estar en uno de tres estados: pendiente (pending), cumplida (fulfilled) o rechazada (rejected). Usar promesas ayuda a evitar el "infierno de callbacks" y facilita la gestión de operaciones asíncronas, mejorando la claridad y mantenibilidad del código.

Async

La palabra clave async se utiliza en programación para definir funciones asíncronas que permiten escribir código asíncrono de manera más simple y legible. Las funciones marcadas con async devuelven automáticamente una promesa (en JavaScript) o un objeto de futuro (en Python). Junto con await, que se usa dentro de una función asíncrona para esperar la resolución de promesas u operaciones asíncronas, async permite escribir código que se comporta de manera síncrona, facilitando la lectura y evitando el "infierno de callbacks".

Await

La palabra clave await se usa en programación dentro de funciones asíncronas definidas con async para pausar la ejecución de la función hasta que una promesa (o una operación asíncrona) se resuelva. Esto permite escribir código asíncrono de manera que se lea y se comporte de forma más síncrona, mejorando la claridad y la legibilidad del código. await facilita la gestión de operaciones asíncronas al evitar la anidación excesiva de callbacks y promesas.
